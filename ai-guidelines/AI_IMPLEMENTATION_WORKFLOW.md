# AI Implementation Workflow

You must follow this workflow strictly.

---

# PHASE 1 — ANALYSIS & DOCUMENTATION

Before redesigning any screen:

Create documentation files inside:

docs/ui-revamp/

Required files:

1. global-ux-audit.md
2. design-system-strategy.md
3. layout-architecture.md
4. revamp-roadmap.md

Do NOT begin redesign implementation yet.

---

# REQUIRED DOCUMENTATION

## 1. Global UX Audit

Analyze:

- Existing UX issues
- Layout inconsistencies
- Workflow inefficiencies
- Space utilization problems
- Scrolling problems
- Data visibility issues
- Navigation issues
- Component inconsistencies

---

## 2. Design System Strategy

Define:

- Spacing system
- Grid/layout rules
- Card patterns
- Table patterns
- Form patterns
- Toolbar patterns
- Drawer/modal patterns
- Typography hierarchy
- Tab system
- Responsive behavior
- Sticky behavior standards

---

## 3. Layout Architecture

Define:

- Global navigation structure
- Workspace layout system
- Tab architecture
- Split-panel strategies
- Detail-view strategies
- Dashboard structures
- Content density rules

---

## 4. Revamp Roadmap

Define:

- Suggested redesign order
- Shared reusable components
- Dependencies between modules
- Migration strategy
- Risk areas
- UX consistency checkpoints

---

# PHASE 2 — WAIT FOR PAGE INSTRUCTIONS

After documentation is complete:

STOP.

Wait for user to provide:

- Page/module names
- Screenshots
- Workflows
- Specific redesign requests

Do NOT redesign the entire application automatically.

---

# PHASE 3 — PAGE-BY-PAGE REDESIGN

For every page redesign:

First perform analysis.

Provide:

## Current UX Problems

- Clutter
- Scrolling issues
- Visibility issues
- Workflow inefficiencies
- Poor spacing
- Poor hierarchy

## Proposed Improvements

- New layout structure
- Tab structure
- Information grouping
- Sticky actions
- Drawer usage
- Better workflows

Only after analysis:
Begin implementation.

---

# IMPLEMENTATION EXPECTATIONS

All code must be:

- Reusable
- Modular
- Scalable
- Consistent

Prefer reusable:

- Layout wrappers
- Table components
- Toolbar components
- Filter components
- Drawer systems
- Form sections
- Tab systems

Avoid one-off styling and inconsistent layouts.

---

# CONSISTENCY RULE

Every redesigned page must visually and structurally align with previously redesigned pages.

Maintain:

- Spacing consistency
- Layout consistency
- Component behavior consistency
- Navigation consistency
- Interaction consistency

---

# IMPORTANT RULE

Never redesign unrelated pages.

Only work on the page/module explicitly requested by the user.
