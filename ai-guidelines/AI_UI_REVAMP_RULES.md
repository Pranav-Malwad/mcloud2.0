# Enterprise UI/UX Revamp Master Rules

You are redesigning and modernizing an enterprise SaaS application.

Your responsibility is to improve the application's UI/UX while preserving:

- Existing business workflows
- Existing theme colors
- Existing typography
- Existing branding identity

The redesign must feel modern, premium, enterprise-grade, compact, and highly workflow efficient.

The application should feel inspired by:

- Linear
- Jira
- Modern Salesforce
- Notion
- Enterprise CRM dashboards
- Modern admin platforms

---

# PRIMARY OBJECTIVES

The redesign must prioritize:

- Fast workflows
- High information visibility
- Reduced user effort
- Better scanability
- Reduced scrolling
- Better organization
- Better spacing alignment
- Better visual hierarchy
- Consistent UX patterns

The application should support power users working with large datasets and repetitive workflows.

---

# GLOBAL UX PRINCIPLES

## 1. Minimize Vertical Scrolling

Avoid long stacked layouts.

Users should see important information immediately.

Prefer:

- Tabs
- Split panels
- Side drawers
- Accordions
- Expandable sections
- Compact cards
- Multi-column forms
- Sticky action bars
- Sticky filters
- Sticky table headers

Avoid:

- Long single-column pages
- Excessive whitespace
- Large empty sections
- Oversized cards

---

## 2. Dashboard-Oriented Experience

The application should feel operational and dashboard-driven.

Each screen should maximize:

- Visibility
- Workflow efficiency
- Fast interactions
- Context awareness

Prefer layouts with:

- Toolbars
- Action panels
- Context sections
- Resizable panels
- Dense but readable information
- Inline interactions

---

## 3. Tab-Based Workflow Architecture

The application should use structured navigation.

Prefer:

- Horizontal tabs
- Left-side navigation tabs
- Nested tabs
- Detail tabs
- Workspace tabs

Avoid deep navigation chains and unnecessary page switches.

---

## 4. Enterprise Visual Design

The UI must feel:

- Professional
- Lightweight
- Sharp
- Minimal
- Modern
- Clean

Use:

- Subtle shadows
- Soft borders
- Crisp spacing
- Compact layouts
- Clear typography hierarchy

Avoid:

- Flashy design
- Heavy gradients
- Decorative UI
- Excessive animations
- Bright random colors

---

## 5. Data Visibility Optimization

Data-heavy workflows should prioritize visibility.

Tables/grids must support:

- Sticky headers
- Compact readable rows
- Better alignment
- Expandable rows
- Side detail drawers
- Inline actions
- Better filtering
- Better search UX

Avoid forcing users into multiple screens for small details.

---

## 6. Form Experience Standards

Forms must be redesigned for operational efficiency.

Requirements:

- Multi-column forms
- Logical grouping
- Collapsible sections
- Accordion segmentation
- Sticky action buttons
- Inline validation
- Compact field spacing

Avoid very long vertical forms.

---

## 7. Responsive Behavior

Primary focus:

- Desktop-first enterprise workflows

Large screens must utilize width efficiently.

Avoid excessive empty space.

Tablet layouts should remain usable.

---

# DESIGN CONSTRAINTS

## MUST PRESERVE

- Existing brand colors
- Existing typography/font family
- Existing workflows
- Existing business logic
- Existing terminology

## MUST NOT

- Introduce random theme colors
- Create flashy UI
- Add unnecessary animations
- Increase whitespace unnecessarily
- Overcomplicate workflows

---

# LAYOUT EXPECTATIONS

Every redesigned page should prefer:

- Dashboard layouts
- Split layouts
- Side-by-side sections
- Compact cards
- Dense but readable tables
- Sticky toolbars
- Context panels
- Resizable sections
- Side drawers
- Tab-based segmentation

Avoid long vertically stacked sections.

---

# VISUAL HIERARCHY RULES

Prioritize:

1. Important actions
2. Important metrics
3. Important data
4. Filters/search
5. Secondary details

Important actions should remain visible without scrolling whenever possible.

---

# UX QUALITY EXPECTATIONS

Every redesign should improve:

- Scanability
- Readability
- Workflow speed
- Context switching
- Information grouping
- Space utilization
- Navigation clarity

The application should feel enterprise-grade and operationally efficient.

---

# IMPLEMENTATION RULES

When redesigning a page:

1. Analyze current UX issues first
2. Explain problems clearly
3. Explain redesign strategy
4. Then implement redesign
5. Maintain consistency with previous redesigned pages

Never redesign unrelated screens unless explicitly instructed.

---

# FINAL EXPERIENCE GOAL

The final product should feel like:

"A modern enterprise SaaS platform optimized for high-speed operational workflows with maximum information visibility and minimal scrolling."
